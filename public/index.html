What is REST?

What is REST?

  • A preferred architectural style for APIs
  • Focus is on interactions between the client and server (rather than implementation details)
  • Improves performance, scalability, simplicity, modifiability, visibility, portability, and reliability

More about REST

  • REpresentational State Transfer - described by Roy Fielding in his doctoral dissertation in 2000
  • Not dependent on any protocol, but almost every RESTful service uses HTTP

HATEOAS

Hypermedia as the Engine of Application State

  • Response must contain actions that client can take based on the request
    • Allows clients to discover the structure of the API
  • Commonly in a standardized structure like json api or HAL
  • Allows clients to create and update themselves

What are the 5 RESTful routes of CRUD?

The 5 RESTful Routes of CRUD

  • app.get('/') - LIST - Get all
  • app.get('/:id') - READ - Get one
  • app.post('/') - CREATE - Post one
  • app.patch('/:id') - UPDATE - Update one
  • app.delete('/:id') - DELETE - Delete one

Are REST and CRUD the same thing?

Nope.

REST is an architectural style for designing APIs and describes interaction between the CLIENT and SERVER

CRUD is a database concept that describes actions on data

They work together and are often conflated, but they are not the same thing.
